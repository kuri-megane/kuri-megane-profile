## 自己PR

### スピードと品質を両立する開発推進力

私は物事をひとつずつ整理しながら進めることを得意としており、おさえるポイントを担保しながらスピード感を持って開発を進めることを意識しています。
要件定義からリリース後の運用まで一貫して担当することもあり、その中でも仕様を明確に表現したテスト設計を重視しています。
Webシステムのリプレイスでは、不具合が多発する中での参加でしたが、社内初となるデータベース操作の担保に着目した結合テストとシナリオテストの提案により、予定より1ヶ月早く不具合0件でリリースしました。
現職でもQAと連携したテスト設計の仕組み化を主導し、エンジニア側から開発初期にQAを巻き込むことで、月3〜4回ペースのリリースを品質を落とさず継続しています。
機能リプレイスでは、置き換え対象の既存処理について先行してユニットテストを整備し、置き換え後の処理が同じテストをクリアすることを完了条件と定め、仕様の暗黙知を引き継ぎながら意図しないデグレを防止しています。
また本番障害が発生した際も、根本原因を特定してから修正を適用するアプローチを徹底し、再発防止まで含めて対応しています。

### ユーザーからのフィードバックを反映する力

ユーザーを常に意識し、フロントエンドや営業メンバーと密にコミュニケーションを取って進めています。
ナビゲーション位置推定精度向上の研究開発では、ユーザーヒアリング実施の提案によりユーザーが困っている場面から重視する点や指標を決めたことで、精度やユーザー満足度も向上し、課題の本質の追求に繋がりました。
設備点検アプリケーションでは、点検現場の見学を通じて、不便な点や現場環境を考慮した仕様検討や改善により、より多く使っていただける機能の提供ができました。
また、カスタマーサポートからの問い合わせデータや本番ログを定期的に確認することで、ユーザー影響のある潜在的な問題を先回りして対処することを継続しています。

### 気づきからプロセスを動かす力

自分が気づいたプロセスや体制上の改善点を起点に、チームを動かすことを得意としています。
障害が多発した局面やチーム立ち上げ期にはレトロスペクティブの仕組みを自ら導入し、継続的な改善サイクルが回るよう働きかけました。
汎用設備点検プロジェクトでは、リリースフローおよびブランチ戦略の見直しにより、本番に出せない差分の確認やリバート対応など開発起因でリリース準備に時間がかかっていた部分を約1/4に短縮しました。
QAやPMの負荷が高まったタイミングでは、職種の境界に縛られず業務を巻き取ってきました。生成AIを活用したテストケース作成、カスタマーサポート向けリリースノートやリリース判定書の作成、QAテストそのものの実施まで対応し、リリースや顧客対応のボトルネックを解消しています。
日々の開発・運用で気付いた品質・体制上の課題は放置せず、QA・PM・CSなど該当するロールのメンバーを巻き込んで具体的な改善まで進めています。

### 生成AIを活用した業務拡張と組織知の体系化

GitHub CopilotやClaude Codeなどの生成AIをコーディング支援にとどめず、業務全般に取り入れて活用しています。
日々の開発では、コーディングだけでなく、本番障害発生時の調査支援、リリース準備時のチェック、設計書作成の補助などに組み込み、開発スピードと品質の両立に役立てています。
さらに、ミーティング議事録やSlackの議論ログを起点として組織知の整備にも取り組んでいます。新規参画者向けのドメイン用語集の自動生成、過去のバグやインシデントを再発させないためのチェックリスト、顧客カルテや顧客ごとの非機能要件リストなどを作成し、設計・仕様検討フェーズへのフィードバックとして還元しています。
ツールを使いこなすこと自体を目的化せず、得られたアウトプットを業務プロセスに組み込んで再現性のある成果につなげることを重視しています。

開発業務を主軸とし、フロントエンドからバックエンド・クラウドインフラまで全レイヤーにわたる開発を担いながら、品質保証・リリース管理・障害対応まで一貫して推進することに価値発揮の軸を置きます。
ユーザーとの対話に労力を惜しまず、ユーザー自身が言語化しきれていない課題まで深く理解した上で、実装や運用の現実を踏まえた最善の実装と改善に取り組むことを重視します。

将来的には、現職での顧客現場訪問・カスタマーサポート連携・仕様検討の経験を足がかりに、要件定義や顧客課題の深掘りといった上流工程にも領域を広げていきたいと考えています。

#### AIスポット撮影・他社製飛行計画ファイル取り込み機能の開発

- 時期: 2024年10月〜2025年4月(6か月)
- 概要
  - **AIスポット撮影**: DJI製AI撮影機能を活用し、事前登録した撮影対象の画像を基にカメラの向きを自動調整して被写体を探索・撮影する機能。高ズームでも被写体を漏らすことなく精密撮影でき、固定ルートでは困難だった損傷・劣化部位の高精度記録を実現。
  - **他社製飛行計画ファイル取り込み**: DJI Pilot 2など外部ソフトウェアで作成した飛行計画ファイルを自社プラットフォームへ直接取り込む機能。機体の送信機に依存せず複数機体でのルート共有・クラウド一元管理を実現し、顧客現場の導入障壁を大幅に引き下げ。
- 担当業務
  - 要件定義・仕様検討・設計(顧客ユースケース分析・要件整理を含む)
  - 既存の「クラウドからドローンへ」のデータ連携とは逆方向となる「ドローン・外部ツールからクラウドへの」取り込み機能拡張において、各サービスへの影響を分析した上で拡張設計を主導しました
  - 外部ファイルの保持するデータ項目が既存システムの10倍超に及ぶ点について、製品方針(主要機能の明快さを優先)に基づき将来的にも画面描画に使用しない項目をデータベースのカラム追加ではなく拡張JSON形式での保持を採用し、開発期間を圧縮しました
  - 「取り込み前後の飛行計画ファイル差分ゼロ」を検証合格基準として制定し、差分がなければ飛行検証を省略できる体制を構築することで、膨大な飛行検証コストを削減しました
  - 既存処理を含む単体テスト・APIテストの大幅強化(バックエンドを共用する他製品への意図しない影響の防止)
- 成果・実績
  - AIスポット撮影：複数顧客のPoC継続・本格導入判断を後押ししました
  - 飛行計画ファイル取り込み：競合製品との差別化要因となり、複数顧客のプラットフォーム採用の決め手となりました
